# meta-ai-organizer

A meta repository for tracking and automating work carried out by AI agents across Canonical Kubernetes repositories. All artefacts — scripts, reports, prompts, and project state — are version-controlled here so every action is auditable and reproducible.

## Repository Structure

```
meta-ai-organizer/
├── .github/
│   ├── prompts/       # All reusable prompts (roles + workflows) — single source of truth
│   ├── agents/        # @-mentionable agents (thin wrappers → prompts/)
│   ├── chat-modes/    # Conversation-wide modes (thin wrappers → prompts/)
│   └── copilot-instructions.md
├── Repositories/      # Machine-readable list of all GitHub repos
├── projects/          # Active projects with context, scripts, and progress tracking
├── scripts/           # Standalone automation scripts
└── reports/           # Output reports documenting agent runs and findings
```

### `projects/`

Each project tracks a multi-repo campaign or initiative:

```
projects/<name>/
  README.md           # Project overview, goals, Jira ticket
  PROGRESS.yaml       # Per-repo status tracking
  CHANGELOG.md        # Dated log of significant actions, PRs, decisions
  SUMMARY.md          # High-level project summary (updated at milestones)
  context/            # Reference docs, security policies, codebase notes
  scripts/            # ALL scripts used during the project (any language)
```

### `.github/prompts/`

Single source of truth for all AI agent prompts. Invoke via `/name` in Copilot Chat. See [`.github/prompts/README.md`](.github/prompts/README.md).

### `scripts/`
Standalone automation scripts not tied to a specific project. See [`scripts/README.md`](scripts/README.md).

### `reports/`
Markdown reports generated by AI agents. See [`reports/README.md`](reports/README.md).

## Getting Started

1. **Start a project** — create a directory under `projects/` with the required structure (README, PROGRESS.yaml, CHANGELOG.md, SUMMARY.md, context/, scripts/).
2. **Use prompts** — type `/batch-processing`, `/enumerate-repos`, `/resume-session`, or any role (`/reviewer`, `/developer`, etc.) in Copilot Chat.
3. **Persist scripts** — every script the LLM generates must be saved in the project's `scripts/` directory, regardless of language.
